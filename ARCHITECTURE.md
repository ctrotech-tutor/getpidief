## Inngest Workers Summary

| Worker | Trigger | Schedule | Purpose |
|---|---|---|---|
| `processUploadedDocument` | `document/uploaded` | Event-driven | PDF metadata, thumbnail, search vector, dedup |
| `handleDocumentApproved` | `document/approved` | Event-driven | Cache invalidation, trending set, counts, notify followers |
| `handleDocumentRejected` | `document/rejected` | Event-driven | Notify contributor, reputation penalty |
| `recalculateTrending` | Cron | Every 15 min | Score documents, rebuild Redis sorted sets |
| `flushEngagementCounters` | Cron | Every 5 min | Buffer views/downloads/likes → DB batch update |
| `rebuildSearchVectors` | `system/rebuild-search-vectors` | Event-driven | Rebuild tsvector with tags |
| `updateReputation` | `reputation/update` | Event-driven | Points ledger, level update, Redis leaderboard |
| `checkAndAwardBadges` | `reputation/check-badges` | Event-driven | Evaluate all badge criteria, award + notify |
| `updateUserStreak` | `user/streak-update` | Event-driven (debounced 1h) | Daily activity streak, streak bonus |
| `sendNotification` | `notification/send` | Event-driven | In-app + email + push delivery |
| `sendWeeklyDigest` | Cron | Monday 9AM UTC | Fan-out digest emails to all eligible users |
| `processWeeklyDigest` | `email/weekly-digest` | Event-driven | Compile + send individual digest |
| `takeDailySnapshot` | Cron | Midnight UTC | Analytics aggregation to DB |
| `aggregatePopularSearches` | Cron | Hourly | Search logs → Redis sorted set |
| `updateInstitutionLeaderboard` | Cron | Every 6h | Institution activity scores |
| `cleanupExpiredData` | Cron | 3AM daily | Tokens, soft-deleted docs, old logs |

## Redis Key Architecture

```
cache:document:{slug}              ← Full document with relations (5min TTL)
cache:user:{username}              ← Public user profile (2min TTL)
cache:categories:all               ← All active categories (1hr TTL)
cache:tags:popular:{n}             ← Top tags by usage (30min TTL)
cache:institutions:popular         ← Top institutions for onboarding (1hr TTL)
cache:recommended:{userId}         ← Personalized doc recommendations (30min TTL)

trending:documents:global          ← ZSET {documentId: score}
trending:documents:cat:{id}        ← ZSET {documentId: score} per category
trending:documents:inst:{id}       ← ZSET {documentId: score} per institution
trending:tags:global               ← ZSET {tagId: score}
trending:contributors:global       ← ZSET {userId: reputationScore}
trending:institutions:leaderboard  ← ZSET {institutionId: activityScore}

search:popular:global              ← ZSET {query: count}
recent:viewed:{userId}             ← ZSET {documentId: timestamp} max 50
recent:searches:{userId}           ← ZSET {query: timestamp} max 20
```
